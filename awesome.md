# Awesome 宣言

如果您希望您的清單被包含在 `awesome` 中，請嘗試在您的清單中僅包含真正優秀（awesome）的東西。畢竟，這是一份精選（curation），而不是一份收集（collection）。

但是**什麼是 Awesome？**

## 只有真正優秀的才是 Awesome

研究您包含的內容是否真的非常出色。只將您或其他貢獻者可以親自推薦的內容列入清單。您寧願漏掉一些內容，也不要包含過多平庸的東西。

## Awesome 徽章 (Badge)

此徽章適用於 Awesome 清單。

[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat2.svg)](https://awesome.re)

在您清單的頂部，緊鄰標題的位置添加一個 Awesome 徽章。[範例](https://github.com/sindresorhus/awesome-nodejs)。您可以選擇普通徽章或扁平化徽章。

```md
[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat.svg)](https://awesome.re)
[![Awesome](https://awesome.re/badge-flat2.svg)](https://awesome.re)
```

您可以在未包含在此處的清單以及私有清單上使用該徽章。

*不應以任何方式修改徽章。*

## Awesome Mentioned 徽章

此徽章旨在用於出現在 Awesome 清單中的項目（**而非 Awesome 清單本身**）。例如，[Chalk](https://github.com/chalk/chalk) 項目可以使用此徽章，因為它被列在 [Awesome Node.js](https://github.com/sindresorhus/awesome-nodejs) 中。對於項目來說，這完全是可選的，但這是展示它們已被列入 Awesome 清單的一種很好的方式。您可以選擇普通徽章或扁平化徽章。

[![Mentioned in Awesome](https://awesome.re/mentioned-badge.svg)](https://awesome.re)
[![Mentioned in Awesome](https://awesome.re/mentioned-badge-flat.svg)](https://awesome.re)

**填寫佔位符（名稱與 URL）：**

```md
[![Mentioned in Awesome <在此插入清單名稱>](https://awesome.re/mentioned-badge.svg)](https://github.com/<在此插入清單 URL>)
[![Mentioned in Awesome <在此插入清單名稱>](https://awesome.re/mentioned-badge-flat.svg)](https://github.com/<在此插入清單 URL>)
```

範例：

```md
[![Mentioned in Awesome Node.js](https://awesome.re/mentioned-badge.svg)](https://github.com/sindresorhus/awesome-nodejs)
```

如果您是 Awesome 清單的維護者，您可以鼓勵清單中的項目添加該徽章。

*不應以任何方式修改徽章。*

## 說明為什麼某樣東西是 Awesome 的

除了建議清單中的特定項目外，您還應該告知讀者它*為什麼*在清單中，以及他們將如何從中受益。

## 明確清單的主題

在您的 Readme 頂部有一個簡潔的描述。確保您的清單涵蓋一定的範圍，而不是其他任何東西。如果您認為其他 Awesome 清單已經很好地涵蓋了某個主題，請連結到它們。

## 注意語法

確保您的清單語法正確、沒有錯別字且沒有 Markdown 格式錯誤。這也應適用於 Pull Request。

## 選擇適當的授權條款 (License)

請記住，如果您[尚未選擇授權](https://choosealicense.com/no-license/)，這基本上意味著人們*不被允許*複製、分發或建立衍生作品。

[創用 CC (Creative Commons) 授權](https://creativecommons.org/)非常適合此目的。**我們建議使用 [`CC0`](https://creativecommons.org/publicdomain/zero/1.0/)。**不建議使用 MIT、BSD、GPL 等代碼授權。

## 包含貢獻指南

為您的清單做出貢獻的人應該清楚地了解他們應該如何做。

如果您不想從頭開始寫一份，可以隨時使用我們的 [contributing.md](contributing.md) 並根據您自己的需求進行修改。

## 正確地美化您的清單

建立[目錄 (Table of Contents)](https://github.com/sindresorhus/stuff/blob/main/toc-generators.md)，將內容組織到不同的類別中，並在合適的情況下使用圖像。確保所有條目保持一致（例如，所有條目描述都以 `.` 結尾）。

## 接受他人的意見

如果您是清單的所有者，請尊重他人的意見。如果有許多用戶不同意您的決定，請再三考慮。
