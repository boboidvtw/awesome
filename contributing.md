# 貢獻指南

請注意，此專案發佈時附帶了[貢獻者行為準則](code-of-conduct.md)。參與本專案即表示您同意遵守其條款。

## 添加一個 Awesome 清單

請確保您的 Pull Request 符合[指南列表](pull_request_template.md)。

## 建立您自己的 Awesome 清單

要建立您自己的清單，請查看[說明](create-list.md)。

## 向 Awesome 清單添加內容

如果您有很棒的東西要貢獻給某個 Awesome 清單，這就是您要做的方式。

您需要一個 [GitHub 帳號](https://github.com/join)！如果您是 GitHub 新手，請[從這裡開始](https://github.com/firstcontributions/first-contributions)。

1. 訪問 Awesome 清單的 GitHub 頁面。例如：https://github.com/sindresorhus/awesome
2. 點擊 `readme.md` 文件：![步驟 2 點擊 Readme.md](https://cloud.githubusercontent.com/assets/170270/9402920/53a7e3ea-480c-11e5-9d81-aecf64be55eb.png)
3. 現在點擊編輯圖標。![步驟 3 - 點擊編輯](https://cloud.githubusercontent.com/assets/170270/9402927/6506af22-480c-11e5-8c18-7ea823530099.png)
4. 您可以開始在瀏覽器內編輯器中編輯文件的文本。確保您遵守上述指南。您可以使用 [GitHub Flavored Markdown](https://help.github.com/articles/github-flavored-markdown/)。![步驟 4 - 編輯文件](https://cloud.githubusercontent.com/assets/170270/9402932/7301c3a0-480c-11e5-81f5-7e343b71674f.png)
5. 說明您為什麼提議這些更改，然後點擊 "Propose file change"。![步驟 5 - 提議變更](https://cloud.githubusercontent.com/assets/170270/9402937/7dd0652a-480c-11e5-9138-bd14244593d5.png)
6. 提交 [Pull Request](https://help.github.com/articles/using-pull-requests/)！

## 更新您的 Pull Request

有時，Awesome 清單的維護者會要求您在包含之前編輯您的 Pull Request。這通常是由於拼寫錯誤，或者因為您的 PR 不符合 awesome-* 清單指南。

[這裡](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md)有一篇關於如何更改 Pull Request 以及您可以採取的不同方式的文章。
